Private Mission

하나로 협곡의 방어구 제작소에서는 매일같이 수많은 전투 장비와 부품들이 생산되고 있습니다. 특히 방어구 외부를 보호하는 금속 코팅 작업, 즉 전기 도금 처리는 제품의 품질과 내구성에 직접적인 영향을 미치기에, 매우 정밀한 공정이 요구됩니다.

🚨 품질관리 본부 긴급 퀘스트 발령
최근 열풍 건조 공정 중 일부 제품에서 이상 징후가 발견되었습니다!
공정 상태를 분석하고 품질 문제를 사전에 예방해야 합니다!

이 퀘스트의 목적은, 도금 처리 이후 물기를 제거하는 열풍 공정에서 발생하는 이상 데이터를 탐지하고, 품질 문제를 사전에 예방할 수 있도록 공정 상태를 분석하는 것입니다.

📁 협곡에 전달된 두 종류의 데이터

· PLC(Programmable Logic Controller) — 각 Lot별 오류 정보
· Sound Sensor — 공정 중 발생하는 음향 정보

이상한 소음, 예외적인 진동 패턴, 온도 불균형, 반복적으로 발생하는 오류 Lot들은
공정 내 미세한 이상 징후의 표현일 수 있습니다.

공정의 열풍 건조 과정은 도금된 제품에 남아 있는 수분을 완전히 제거하는 단계입니다. 이 단계에서 문제가 생기면, 수분이 금속 내부에 잔존하여 산화 또는 도금 박리와 같은 심각한 문제로 이어질 수 있습니다.

    43_Sound_Data.zip 파일의 압축을 풀어, 폴더 내 hda-sensor 이름이 붙은 모든 데이터를 가져와, 병합하여 df_sound로 선언하시오.
    • Row 51,084 / Column 6
  2. 2
    Error Lot list.csv파일을 불러와, 날짜와 오류 Lot 번호, Label값으로 변환하고자 한다. 아래와 같이 데이터를 변환한 뒤, Label 항목에 1값 으로만 채워진 Column을 선언하여 데이터 프레임을 df_error로 선언하시오.
    • Row 111 / Column 3
  3. 3
    처리한 두 데이터를 공통된 'Date'와 'Process'에 대해 병합하고자 한다. 'Date'항목과 'Process'항목을 이용해 'ID' Column을 생성하고, 두 데이터를 병합하여 df1으로 선언하시오.
    • 데이터 병합 시, df_sound Table에 맞춰 병합
    • 중복된 Column은 제거
    • Row 51,084 / Column 8
  4. 4
    병합된 df1 데이터의 Label 항목에서 결측값을 모두 0으로 채워 넣은 뒤, 오류가 발생한 1값과 오류가 발생하지 않은 0값의 빈도수를 확인하시오.
  5. 5
    'Label'(오류여부)에 대해 'Temp'(공정온도)의 분포의 차이를 확인하고자 한다. 오류에 따른 공정온도의 분포를 Box Plot으로 시각화 하시오.
  6. 6
    SPC Chart를 시각화 하고자 한다. 21년 9월 7일에 발생한 데이터의 'Index' 값이 1에서 200까지의 데이터를 뽑아, 'Current'값에 대한 관리도 SPC Chart를 시각화 하시오.
    • UCL (Upper Control Limit) : Current 평균에서 표준편차 3배 큰 구간
    • LCL (Lower Control Limit) : Current 평균에서 표준편차 3배 작은 구간
    • 각 데이터 Point는 선으로 연결되어 있어야 하며, X축은 Index Y축은 Current로 구성
